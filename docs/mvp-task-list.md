## 1. MVP 目标

MVP 目标是完成“销售订单需求邮件 -> 智能识别和抽取 -> 商务初审 -> 低风险生产任务单自动发送 / 高风险人工确认 -> 生产疑问转销售 -> 销售答复后重新下达 -> 生产确认关闭 -> 周报统计”的核心闭环。

MVP 默认策略：

1. 初审通过、字段完整、路由唯一、无风险命中的生产任务单自动发送。
2. 低置信、字段冲突、附件解析失败、路由冲突、变更/取消、已排产后修改全部转人工。
3. 只接入一个商务共享邮箱。
4. 至少支持一套生产部门路由规则。
5. 优先支持腾讯企业邮箱账号密码接入、独立系统发件邮箱、邮件正文、HTML、Word、结构化 Excel、ZIP 解包和可解析文本 PDF。
6. 支持外部模型服务可配置。
7. 有效邮件默认永久保留，并提供管理员清理功能。
8. 周报支持导出 PDF，Excel/CSV 明细作为可选导出。

## 2. 交付边界

### 2.1 MVP 必须交付

1. 基础后端、前端、数据库、队列、对象存储和部署脚本。
2. 邮件收取、解析、去重、线程关联和发送。
3. 邮件分类、订单字段抽取、字段证据和置信度。
4. 初审规则、路由规则、异常队列。
5. 任务单自动发送准入和人工确认发送。
6. 生产疑问、销售答复、重新下达、生产确认排产闭环。
7. 订单变更/取消识别和人工分流。
8. 周报统计和发送。
9. 审计日志、关键指标、失败重试和幂等控制。
10. MVP 验收样本测试。

### 2.2 MVP 不交付

1. ERP、CRM、MES、OA 对接。
2. 自动发送生产任务单。
3. 复杂 OCR 高准确率处理。
4. 多生产基地复杂路由编排。
5. 移动端审批和 IM 通知。
6. 完整自助式规则 DSL。
7. 生产排程和交期承诺。
8. 钉钉和企业微信 SSO 正式对接，仅预留接口。

## 5. 建议里程碑

### M0. 工程骨架与基础设施

目标：项目可启动、可建库、可登录、可配置邮箱。

包含任务：E0、E1。

退出标准：

1. 本地 Docker Compose 可启动。
2. 管理员可登录后台。
3. 数据库迁移可重复执行。
4. 邮箱连接配置可保存并测试。

### M1. 邮件接入与解析

目标：系统可以读取真实邮件并保存原文、正文、附件。

包含任务：E2、E3。

退出标准：

1. 指定邮箱新邮件 1 分钟内入库。
2. 重复邮件不会重复入库。
3. 邮件详情可查看正文和附件。
4. Word、Excel、ZIP 内文件和文本 PDF 可解析。

### M2. 分类、抽取、初审与异常

目标：系统可以识别订单邮件、抽取字段、执行初审并分流异常。

包含任务：E4、E5、E6。

退出标准：

1. 标准订单邮件可生成结构化订单需求。
2. 关键字段有置信度和来源证据。
3. 缺字段、风险、路由冲突进入正确队列。
4. 人工可修改字段并重新初审。

### M3. 任务单下达闭环

目标：系统可以生成生产任务单草稿，并由商务确认后发送。

包含任务：E7、E10 部分页面。

退出标准：

1. 初审通过后生成 V1 草稿。
2. 商务负责人可预览、编辑、确认发送。
3. 发送成功后任务进入待生产反馈。
4. 发送失败不会重复下达。

### M4. 生产疑问与销售答复闭环

目标：系统可以处理生产疑问、销售答复和重新下达。

包含任务：E8、E9。

退出标准：

1. 生产疑问可关联任务并转发销售。
2. 销售答复可判断完整性。
3. 答复充分后生成 V2+ 草稿。
4. 生产确认排产后任务关闭。
5. 变更/取消进入正确人工分流。

### M5. 周报、审计与验收

目标：系统可以自动生成周报并通过 MVP 验收。

包含任务：E11、E12、E13。

退出标准：

1. 周报按配置发送，可导出 PDF，汇总和明细一致。
2. 所有关键动作有审计记录。
3. 验收样本指标达到 PRD 18.3。
4. 形成上线问题清单和 V1 待办。
