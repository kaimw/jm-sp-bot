## 二、🔔 通知中心（红点+右侧抽屉 — 替代原有的跑马灯和通知条）

### 2.1 交互流程

```
异常产生 → 顶部栏🔔图标出现红点+数字
              │
              ├─ 用户看到红点 → 点击🔔
              │     ↓
              │  右侧滑出通知抽屉（遮罩层覆盖页面）
              │     ↓
              │  显示异常简报列表（按时间倒序）
              │     ↓
              │  点击某条简报 → 跳转到异常中心详情页
              │     ↓
              │  点击"查看全部异常" → 跳转到异常中心列表页
              │
              └─ 用户忽略 → 红点持续存在
                    ↓
                用户处理完异常后 → 红点自动消失
```

### 2.2 顶部栏 🔔 按钮

```
┌────────────────────────────────────────────────────────────┐
│  [J] 商务部小J                    🔔 7   👤 admin  [退出]  │
│                                    ╰─╯                    │
│                                   红底白字圆角             │
│                                   显示待处理异常数         │
└────────────────────────────────────────────────────────────┘
```

**行为：**
- 无异常时：🔔 无红点（静默）
- 有异常时：🔔 右上角红色圆角 badge + 数字
- 点击🔔：右侧滑出通知抽屉页面内容灰化遮罩
- 点击遮罩层或再次点击🔔：关闭抽屉

```
**宽度：** 420px
**位置：** 从右侧滑出，页面主体灰化遮罩
**动画：** 滑入 0.25s ease-out
**每条简报包含：** 异常图标 + 类型 + 级别badge + 订单号 + 摘要 + 时间 + [查看]按钮
**点击"查看"：** 关闭抽屉 + 跳转到异常中心并打开该异常详情
**点击"查看全部异常"：** 关闭抽屉 + 跳转到异常中心列表页
```

### 2.4 与现有跑马灯的切换

| 现有 | 替换为 |
|------|--------|
| `global-exception-ticker`（页面顶部跑马灯） | **移除**，改为🔔红点 |
| `notice-trail`（页面内通知条） | **移除**，由🔔+Toast替代 |
| 紧急异常自动弹Toast（红色，手动关闭） | **保留**（Critical级别异常仍需即时提醒） |
| 普通通知自动弹Toast（绿色，自动消失） | **保留**（制单成功等） |

**保留 Toast 的规则：**
- Critical 级别异常 → 红色 Toast + 🔔红点（双重提醒）
- 普通通知 → 绿色 Toast 3秒自动消失
- High/Medium 异常 → 仅🔔红点，不弹Toast

## 三、主侧边栏导航（优化后）

按**商务用户的操作流程**重新组织：

```
┌──────────────────────┐
│ J  商务部小J          │  ← Brand 区不变
│     订单中台工作台     │
├──────────────────────┤
│                       │
│ 📊 工作台             │  ← 总览大盘
│   总览 / KPI / 告警   │
│                       │
│ 📦 订单处理            │  ← 核心！最重要的页面放最前面
│   预审 / 制单 / 发货   │
│                       │
│ ⚠️ 异常中心            │  ← 主动查看/处理异常
│   阻断 / 待办 / 诊断   │
│                       │
│ 📨 消息中心            │  ← 邮件发送记录
│   通知 / 留痕 / 状态   │
│                       │
├── ──── ──── ──── ── │  ← 视觉分割线
│                       │
│ 📋 主数据              │  ← 配置类集中在一起
│   物料价格 / 别名      │
│   主体-仓库            │
│   物料例外              │
│                       │
│ 📦 库存管理            │  ← 独立的库存操作
│   查询 / Excel导入 / 导入历史 │
│                       │
├── ──── ──── ──── ── │
│                       │
│ 🔧 设置               │  ← 保持现有 settings 入口
│   系统接入 / 规则 / 通知│
│                       │
└──────────────────────┘
```

**🔔 红点与侧边栏的职责分工：**

| 组件 | 职责 | 
|------|------|
| 🔔 红点 | **被动提醒** — 有异常时主动告诉你，点击直接看简报 |
| ⚠️ 异常中心（侧边栏） | **主动查看** — 你想全面查看和处理异常时进入 |
| 📨 消息中心（侧边栏） | **历史记录** — 查看所有已发送的通知邮件 |

**与当前导航的差异对比：**

| 当前 | 优化后 | 理由 |
|------|--------|------|
| 运营看板 → 总览 / CRM订单 | 📊 工作台 | 整合到一个页面 |
| 商务处理 → 异常接管 / 生产任务 | 📦 订单处理（新增核心页） | 订单处理是日常最高频操作 |
| 履约与资产 → 发货执行 / 物料中心 | ⚠️ 异常中心 / 📨 消息中心 | 保留入口，🔔红点辅助提醒 |
| 运维监控 → 通知队列 / 邮件留痕 / 队列审计 | — | 整合到消息中心和异常中心 |
| — | 📋 主数据（新增） | 集中所有配置映射 |
| — | 📦 库存管理（新增） | 独立库存操作区 |
| Settings（底部入口） | 🔧 设置（位置不变） | 保持现有 antigravity 风格 |
