## 第一部分：当前人工处理的全景（From 脑图 + 调研）

### 1.1 人工处理 7 大环节

根据脑图结构，当前商务（B）每天手工执行的流程：

```
CRM订单
  ↓ ① 信息提取（人工阅读CRM + 邮件附件 + PI）
  ↓ ② 物料映射（CRM名称 → 金蝶料号，人工判断版本/规格）
  ↓ ③ 库存确认（问海外仓/问国内/借货/生产备货）
  ↓ ④ 特殊需求识别（物流备注/生产备注/报关备注）
  ↓ ⑤ ERP制单（在金蝶中手工录入：客户/物料/数量/价格，选主体账号）
  ↓ ⑥ 发发货通知（邮件发送给仓库+物流+生产+销售）
  ↓ ⑦ 跟踪下推（物流发货后邮件通知 → 手动在金蝶中做下推）
```

```
┌──────────────────────────────────────────────────────────────┐
│ ① 感知层：CRM 订单同步                                       │
│   CRM审批完成 → CDP同步 → 幂等入库 → 一期范围过滤 → 入队     │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│ ② 前置审核：商务条件检查（新增 N-009）                       │
│   检查 CRM 合同审批状态 → 未通过 → 阻断 + 通知销售            │
│   检查回款状态 → 异常 → 特殊审批流程                          │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│ ③ 中台建单与预审                                             │
│   upsert_middle_platform_order → 状态机 IMPORTED              │
│   → 11条规则链(含物料别名匹配N-001)                           │
│   → 阻断/通过                                                │
└──────────────────────────────────────────────────────────────┘
                            ↓ (通过)
┌──────────────────────────────────────────────────────────────┐
│ ④ ERP自动制单与提交（N-007 一期核心 🆕）                      │
│   Save(制单) → Submit(提交) → Audit(审核)                    │
│   字段映射: CRM客户→金蝶客户, 物料名→金蝶料号(别名)          │
│   多主体: 根据销售组织选择金蝶账号(深圳/香港)                  │

                            ↓ (成功)

│ ⑤ 发货通知 & OMS 下推                                        │

                            ↓
┌──────────────────────────────────────────────────────────────┐
```

### 5种改单的处理路径

| 场景 | 中台判断 | 处理动作 |
|------|---------|---------|
| 已闭环订单修改 | 状态已归档 | 新建中台订单+新ERP单，旧单保留历史 |
| 未闭环未发货修改 | 状态在VALIDATED-DELIVERY_READY | 撤销发货通知→重审→制单→覆盖原ERP单(Save修改) |
| 时间太久加需求 | 查上次处理时间 | 新增订单，不改原单 |
| 已部分发货再加 | 有部分SHIPPED记录 | 仅新物料走完整流程，已发部分不动 |
| 取消订单 | 未发→撤销；已发→异常 | 未发撤销+ERP作废；已发拦截物流 |
