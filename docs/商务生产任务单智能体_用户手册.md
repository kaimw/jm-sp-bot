# 商务生产任务单智能体用户手册

文档版本：v1.0  
编制日期：2026-04-23  
适用系统：商务生产任务单智能体 MVP  
适用对象：商务负责人、商务操作员、销售人员、生产部门、管理员、运维人员  

## 1. 手册说明

本手册用于指导用户使用“商务生产任务单智能体”完成生产订单邮件接收、订单信息初审、生产任务单下达、生产疑问转销售、销售补充后重新下达、生产确认关闭、周报生成、异常处理和系统配置等工作。

本系统当前实现形态为一个本地 Web 管理台和后台邮件自动处理服务。系统通过腾讯企业邮箱 IMAP/SMTP 接入商务机器人邮箱，自动读取邮件、解析正文和附件、识别业务类型、生成或发送相关邮件，并在管理台中保留任务、邮件、附件、异常、审计和备份记录。

## 2. 系统定位与处理边界

### 2.1 系统定位

系统定位为“商务生产任务单邮件流程智能体”，主要替代或辅助商务人员完成以下重复性工作：

- 自动接收销售订单需求邮件。
- 自动识别订单需求、生产疑问、销售答复、生产确认、订单变更、订单取消和非目标邮件。
- 自动解析邮件正文和 Word、Excel、PDF、ZIP、TXT、CSV 附件。
- 自动抽取客户、产品、数量、期望交期、订单号、销售人员等关键字段。
- 自动执行销售订单初审规则。
- 自动生成生产任务单邮件并进入外发队列。
- 自动将生产疑问转发给销售，并在销售补充后重新下达任务单。
- 自动记录生产确认并关闭任务。
- 自动生成周报、导出 PDF/CSV，并支持邮件发送。
- 保留原始邮件、附件、字段证据、异常和审计记录，便于追溯。

### 2.2 当前边界

- 系统不直接对接 ERP、MES、CRM、OA 等业务系统。
- 系统不自动承诺交期，不做产能判断或排程决策。
- 订单变更、订单取消、字段缺失、生产部门邮箱未配置、低置信或无法关联任务等场景会进入异常队列，需要人工处理。
- 密码、邮箱授权码、模型 API Key 不应写入代码或文档，应通过环境变量或管理台运行期配置维护。
- 当前默认只有本地管理员登录账号；如需多用户、角色权限、SSO，可作为后续版本扩展。

## 4. 登录与访问

### 4.1 访问地址

本地运行默认访问地址：

```text
http://127.0.0.1:8000
```

如使用 Docker 或部署到服务器，请使用服务器实际域名或 IP 加端口访问。

### 4.2 默认登录账号

默认账号：

| 项目 | 默认值 |
| --- | --- |
| 用户名 | admin |
| 密码 | admin |

生产环境应通过环境变量修改默认账号和密码：

| 环境变量 | 说明 |
| --- | --- |
| ADMIN_USERNAME | 管理员用户名 |
| ADMIN_PASSWORD | 管理员密码 |
| AUTH_SECRET | 会话签名密钥 |
| AUTH_SESSION_SECONDS | 登录会话有效期，默认 28800 秒 |

### 4.3 登录步骤

1. 打开系统访问地址。
2. 在登录页输入用户名和密码。
3. 点击“登录”进入工作台。
4. 使用完成后点击右上角“退出”清理会话。

## 6. 首次使用配置

首次使用建议按以下顺序配置：

1. 在“接入”页面配置机器人邮箱、IMAP、SMTP、显示名称和签名。
2. 在“生产邮箱”页面配置生产部门主送邮箱和抄送邮箱。
3. 在“接入”页面配置模型 Provider；如果暂时不使用模型，可关闭 LLM 兜底识别。
4. 在“初审规则”页面确认必填字段和自定义规则。
5. 在“模板”页面检查生产任务单邮件主题和正文模板。
6. 在“周报”页面配置周报主送和抄送人员。
7. 使用“模拟订单”或真实测试邮箱运行一次端到端验证。

| 默认签名 | 系统自动邮件末尾签名 | 积木易搭AI机器人 |

### 6.2 模型服务配置

进入“接入”页面，在“模型配置”区域维护：

| 字段 | 说明 |
| --- | --- |
| 模型标题 | 便于管理员识别该配置 |
| 服务类型 | 当前按 OpenAI 兼容接口调用 |
| 模型名称 | 供应商模型名称 |
| API Base | OpenAI 兼容接口地址 |
| API Key | 接口密钥，留空表示不修改 |

点击“测试模型”可验证当前模型配置是否可用。也可以在“模型对话测试”中输入一句话，系统会使用当前模型配置发起真实对话请求。

### 6.3 生产部门邮箱配置

进入“生产邮箱”页面，维护默认生产部门邮箱：

| 字段 | 说明 |
| --- | --- |
| 部门编码 | 默认可填写 default |
| 部门名称 | 生产部门显示名 |
| 主送邮箱 | 任务单下达的生产部门收件邮箱，可多个 |
| 抄送邮箱 | 任务单下达时抄送邮箱，可多个 |

如果生产邮箱未配置，销售订单即使通过初审也无法下达生产，会进入“异常”页面并标记为 `RoutingMissing`。

### 6.4 初审规则配置

进入“初审规则”页面，可配置：

- 是否启用销售订单初审。
- 必填字段：客户名称、产品/规格、数量、期望交期等。
- 自定义规则：按字段或全文设置包含、不包含、等于、不等于、大于、小于、正则等判断方式。

未通过初审时，系统会生成“订单信息待补充”邮件发给销售，并在异常队列中保留记录。

### 6.5 任务单模板配置

进入“模板”页面，可维护生产任务单主题模板和正文模板。模板支持变量：

| 变量 | 含义 |
| --- | --- |
| `{{task_no}}` | 生产任务单编号 |
| `{{version_no}}` | 当前版本号 |
| `{{customer_name}}` | 客户名称 |
| `{{salesperson_name}}` | 销售人员名称 |
| `{{salesperson_email}}` | 销售邮箱 |
| `{{product_summary}}` | 产品/规格 |
| `{{quantity_text}}` | 数量 |
| `{{expected_delivery_date}}` | 期望交期 |
| `{{bot_signature}}` | 机器人签名 |

保存模板后，新生成的任务单会使用新模板版本；历史任务保留当时生成的邮件内容。

### 6.6 周报收件人配置

进入“周报”页面，维护：

- 周报主送：财务、CEO、销售总监等。
- 周报抄送：商务负责人或其他相关人员。

多个邮箱使用英文逗号分隔。未配置主送时，系统无法生成周报外发邮件。

## 7. 业务流程使用说明

### 7.1 销售提交新订单需求

销售人员通过企业邮箱发送生产订单需求到机器人邮箱。建议邮件中包含以下字段：

| 字段 | 建议写法 |
| --- | --- |
| 客户名称 | 客户名称：测试客户 |
| 产品/规格 | 产品：积木展示架 A1 |
| 数量 | 数量：120 套 |
| 期望交期 | 期望交期：2026-05-20 |
| 订单号 | 订单号：SO-TEST-001 |

系统处理逻辑：

1. Worker 同步邮件并保存原始 EML。
2. 系统解析正文、HTML 和附件。
3. 系统分类为 `SalesOrderRequirement`。
4. 系统抽取订单字段并记录字段证据。
5. 系统执行初审规则。
6. 初审通过且生产邮箱已配置时，系统生成生产任务单版本并创建外发任务。
7. 系统向销售发送收件回执，告知邮件已入库处理。

### 7.2 初审通过后的生产任务单下达

初审通过后，系统生成生产任务单邮件，邮件主题默认包含：

```text
[生产任务单][任务编号][客户名称][产品/规格][版本号]
```

任务单下达给“生产邮箱”页面配置的主送邮箱和抄送邮箱。外发任务可在“外发”页面查看状态。

当前自动 worker 会处理系统自动流程邮件，包括生产任务单、销售回执、补充请求、生产疑问转发、生产确认回执、周报等。管理员也可点击右上角“发送待发邮件”手动触发外发。

### 7.3 初审未通过和销售补充

如果销售订单缺少必填字段或命中风险规则：

1. 任务不会直接下达生产。
2. 系统创建异常记录，通常类型为 `ReviewNeedManual`。
3. 系统向销售发送“订单信息待补充”邮件，列出缺失或不合规原因。
4. 销售回复补充内容后，系统会再次抽取和初审。
5. 通过后自动生成任务单并下达生产。
6. 仍未通过时，系统会继续保留异常并再次提示补充或等待人工处理。

销售回复补充信息时，建议保留原邮件主题或任务/需求编号，并使用明确字段格式，例如：

```text
客户名称：测试客户
产品：积木展示架 A1
数量：120 套
期望交期：2026-05-20
订单号：SO-TEST-001
```

### 7.4 生产确认排产

生产部门收到任务单后，可直接回复邮件确认排产。系统可识别以下类似表达：

- 确认排产
- 安排生产
- 可以生产
- 已排产
- 同意排产
- 同意生产
- 同意安排生产

系统处理逻辑：

1. 将邮件分类为 `ProductionScheduleConfirmation`。
2. 关联对应生产任务。
3. 将任务状态关闭，关闭原因为 `ScheduledConfirmed`。
4. 生成生产确认相关外发邮件。
5. 默认抄送 CEO、销售发起人和商务抄送邮箱。

### 7.5 生产提出疑问或驳回

生产部门可直接回复任务单邮件说明疑问或驳回原因。系统可识别“驳回、疑问、请确认、信息不足”等表达。

系统处理逻辑：

1. 将邮件分类为 `ProductionQuestion`。
2. 将任务状态更新为生产疑问。
3. 记录疑问内容。
4. 自动生成“生产疑问转销售”邮件，发送给对应销售人员。
5. 给生产部门发送疑问收件回执，说明已转销售。

### 7.6 销售答复生产疑问

销售收到生产疑问邮件后，应直接回复并写清楚补充内容。系统识别销售答复后：

1. 记录销售答复。
2. 尝试从答复中提取新的客户、产品、数量、交期或订单号。
3. 生成新版生产任务单，例如 V2、V3。
4. 将新版任务单重新发送给生产部门。
5. 成功重新下达后，向销售发送处理回执。

如果销售答复中包含“不确定、待确认、稍后补充、还没确认”等不完整表达，系统会判定答复不充分并进入异常队列。

### 7.7 订单变更

销售邮件中包含“变更、更改、修改订单”等关键词时，系统会识别为 `OrderChangeRequest`。

处理规则：

- 如果任务尚未确认排产，系统会尝试抽取变更内容并生成新版任务单草稿。
- 如果任务已确认排产或已关闭，系统会进入人工异常，避免无人值守修改已排产订单。
- 如果系统无法识别结构化变更字段，异常详情会提示人工确认邮件变更内容。

### 7.8 订单取消

销售邮件中包含“取消订单、订单取消、取消生产”等关键词时，系统会识别为 `OrderCancelRequest`。

处理规则：

- 系统不会自动取消生产。
- 任务进入取消审核状态。
- 系统创建 `OrderCancelManualReview` 异常。
- 商务负责人需要线下确认后在“异常”页面关闭异常，必要时人工通知生产和销售。

### 7.9 状态查询邮件

销售或生产可以通过邮件询问需求、订单、任务、排产、生产等状态。系统会根据发件人角色生成状态查询回复。

生产部门还可查询待确认任务。建议邮件内容写为：

```text
请查询当前待确认生产任务
```

如果要确认指定任务，建议回复：

```text
确认排产 PT-20260422-0001
```

### 7.10 最大沟通轮数

系统按后台配置限制同一任务生产与销售之间的沟通往返轮数，默认 3 轮。达到上限后：

- 任务会关闭，关闭原因为 `ConversationMaxRounds`。
- 系统通知销售和生产重新发起完整订单需求。
- 系统创建异常记录，便于商务复盘。

## 8. 页面操作详解

### 8.1 工作台

“工作台”显示系统核心指标：

| 指标 | 含义 |
| --- | --- |
| 任务总数 | 系统中生产任务总量 |
| 草稿待确认 | 任务草稿或重发草稿数量 |
| 已下达 | 已下达或已重新下达生产的任务数量 |
| 生产疑问 | 生产提出疑问、等待销售补充的任务数量 |
| 已关闭 | 已确认排产或其他原因关闭的任务数量 |
| 发送失败 | 外发队列中失败邮件数量 |
| 变更/取消 | 订单变更草稿或取消审核数量 |

建议商务负责人每天上班后先查看工作台，重点关注“生产疑问”“发送失败”“变更/取消”和异常队列。

### 8.2 任务

“任务”页面用于查询生产任务单。支持按以下条件过滤：

- 全文搜索：任务号、客户、产品、订单号、销售邮箱。
- 状态。
- 客户。
- 产品。
- 销售邮箱。
- 订单号。
- 交期。

每条任务显示任务编号、客户、销售、订单号、产品、数量、交期和状态。点击“查看工作流”可查看：

- 当前任务工作流步骤。
- 已完成、进行中、待处理节点。
- 任务相关时间线。
- 邮件、疑问、答复、版本和审计摘要。

### 8.3 模拟订单

“模拟订单”用于本地验证流程，不依赖真实 IMAP 收件。填写销售邮箱、主题和正文后点击“生成任务单草稿”，系统会创建一封模拟入库邮件并执行订单流程。

适用场景：

- 验证抽取字段是否正确。
- 验证初审规则是否生效。
- 验证生产邮箱配置是否完整。
- 验证任务单模板变量渲染是否符合预期。

### 8.4 初审规则

“初审规则”分为两个区域。

“初审开关与必填项”用于控制：

- 是否启用销售订单初审。
- 哪些字段必须填写。

“自定义规则”用于新增业务规则。规则字段包括：

| 字段 | 说明 |
| --- | --- |
| 规则名称 | 便于识别规则用途 |
| 审查字段 | 可选择客户、产品、数量、交期、订单号或全文 |
| 判断方式 | 包含、不包含、等于、不等于、大于、小于、正则等 |
| 规则值 | 用于比较的值或正则表达式 |
| 未通过原因 | 发送给销售或展示给商务的原因 |

规则列表支持搜索、启用、停用和删除。修改后会影响新处理的邮件，历史任务保留历史处理结果。

### 8.5 邮件

“邮件”页面展示所有入库和外发相关邮件记录，支持按主题、发件人、分类、任务、方向过滤。

常见分类：

| 分类 | 含义 |
| --- | --- |
| SalesOrderRequirement | 销售提交新生产订单需求 |
| ProductionQuestion | 生产部门疑问或驳回 |
| SalesClarificationReply | 销售答复生产疑问 |
| ProductionScheduleConfirmation | 生产确认排产 |
| OrderChangeRequest | 订单变更 |
| OrderCancelRequest | 订单取消 |
| NonTarget | 非目标邮件 |
| BounceOrAutoReply | 退信或自动回复 |

当任务关联失败时，可在“异常”页面查看对应异常。

### 8.6 外发

“外发”页面展示系统生成的邮件任务，支持按主题、类型、状态、收件人过滤。

常见状态：

| 状态 | 含义 | 操作 |
| --- | --- | --- |
| Pending | 待发送 | 可等待 worker 或手动点击发送 |
| Sent | 已发送 | 无需处理 |
| Failed | 发送失败 | 点击“重试”重新加入待发送队列 |

常见外发类型：

| 类型 | 含义 |
| --- | --- |
| SalesReceiptAck | 销售收件回执 |
| RequirementSupplementRequest | 订单信息待补充 |
| TaskIssue | 首次生产任务单 |
| ProductionQuestionForward | 生产疑问转销售 |
| ProductionQuestionReceipt | 生产疑问收件回执 |
| SalesReplyTaskReissue | 销售答复后的新版任务单 |
| SalesReplyReissueReceipt | 新版任务单发送成功回执 |
| ProductionConfirmed | 生产确认通知 |
| ProductionRejected | 生产驳回通知 |
| WeeklyReport | 周报邮件 |

### 8.7 异常

“异常”页面用于处理无法自动闭环的情况。默认显示 Open 状态异常，支持按状态、级别、类型搜索。

常见异常类型：

| 类型 | 含义 | 建议处理 |
| --- | --- | --- |
| ReviewNeedManual | 初审未通过 | 查看缺失字段或风险原因，等待销售补充或点击补字段 |
| RoutingMissing | 生产邮箱未配置 | 到“生产邮箱”配置主送邮箱后补字段或重跑流程 |
| MailTaskLinkFailed | 邮件无法关联任务 | 人工判断邮件对应任务，必要时线下处理后关闭 |
| NonTarget | 非目标邮件 | 确认无需处理后关闭 |
| IncompleteSalesReply | 销售答复不充分 | 联系销售补充或人工关闭 |
| OrderCancelManualReview | 订单取消需人工审核 | 线下确认后关闭 |
| ScheduledOrderChangeManualReview | 已排产订单变更需人工审核 | 商务负责人确认后处理 |
| ConversationMaxRounds | 沟通轮数达到上限 | 要求销售重新发起完整订单需求 |

异常操作：

- “补字段”：适用于与订单需求关联的异常。可补充客户、产品、数量、交期、订单号、销售邮箱、销售姓名，并选择是否清除风险标记。补齐后系统会尝试恢复流程并生成任务单。
- “关闭”：表示异常已人工处理或无需继续处理。关闭时可填写备注。

### 8.8 周报

“周报”页面支持：

- 周报预览：查看本周、本月、本年统计摘要。
- 生成周报邮件：将周报加入外发队列。
- 导出 PDF：生成带标题的 PDF 周报。
- 导出 CSV：导出周报明细。
- 保存周报收件人：维护主送和抄送。

周报统计口径：

- 本周：北京时间本周一 00:00 到生成时间。
- 本月：北京时间本月 1 日 00:00 到生成时间。
- 本年：北京时间本年 1 月 1 日 00:00 到生成时间。

周报内容包括：

- 各周期需求总数、已确认数、未确认数。
- 已确认产品订单统计。
- 未确认产品订单统计。
- 销售 Top10 统计，包含需求总数和已确认总数。

### 8.9 接入

“接入”页面用于配置运行期集成项，包括：

- 腾讯企业邮箱 IMAP/SMTP。
- 自动 worker 周期和邮箱限频。
- CEO、商务抄送邮箱。
- ZIP 附件处理限制。
- 最大沟通轮数。
- LLM 兜底识别开关。
- 销售和生产测试邮箱。
- 端到端测试。
- 模型 Provider 配置和测试。

端到端测试会使用销售测试邮箱和生产测试邮箱验证真实腾讯企业邮箱收发链路。测试账号密码只通过管理台保存，不应写入代码或文档。

### 8.10 生产邮箱

“生产邮箱”页面用于维护任务单下达地址。系统当前使用启用状态的默认生产部门邮箱。主送为空会导致任务无法下达并进入异常。

### 8.11 模板

“模板”页面用于维护生产任务单邮件。建议修改模板时注意：

- 主题应保留任务编号和版本号，方便后续邮件关联。
- 正文应保留客户、产品、数量、交期和销售邮箱。
- 不要删除必要变量，避免任务单信息不完整。
- 每次保存会生成新模板版本，新任务使用新模板。

### 8.12 运维

“运维”页面包含五类信息。

入库队列：

- 展示待处理或已处理的后台任务。
- 可按任务类型、状态和错误搜索。

附件解析：

- 展示附件文件名、邮件 ID、解析状态、大小和文本预览或错误。
- 支持 Word、Excel、PDF、ZIP、TXT、CSV。

存储占用：

- 展示附件文件数量、已用空间、预算和占用比例。

清理：

- “清理预览”用于查看将被清理的过期非目标邮件和临时记录。
- “执行清理”会真正执行清理。有效订单邮件默认不清理。

备份：

- “立即备份”会生成 SQLite 数据和附件目录 ZIP 备份。
- 备份列表显示类型、状态、时间和存储路径。

审计：

- 展示关键动作，如邮件入库、任务生成、任务下达、异常关闭、周报生成、配置修改等。
- 支持按事件类型、对象类型和操作人过滤。

## 9. 邮件处理规则参考

### 9.1 订单需求识别

系统会优先根据主题和正文关键词识别订单需求。常见关键词包括：

- 生产订单
- 生产需求
- 订单需求
- 下单
- 产品
- 请排产
- 排产

自然语言订单也可识别，例如：

```text
测试客户需要积木展示架 A1 120 套，请安排生产，交期 2026-05-20。
```

### 9.2 风险关键词

当前内置风险关键词包括：

- 客户未确认
- 价格待定
- 先生产后补合同
- 特急
- 加急

命中风险关键词时，初审会失败并进入异常。

### 9.3 附件支持范围

| 附件类型 | 支持情况 | 说明 |
| --- | --- | --- |
| .docx | 支持 | 提取段落和基础表格文本 |
| .xlsx | 支持 | 提取工作表和单元格文本，按数据值读取 |
| .pdf | 支持 | 支持可提取文本 PDF；扫描件可能无法解析 |
| .zip | 支持 | 默认最大 100MB，最多解压 1 层 |
| .txt | 支持 | UTF-8 或可替换解码 |
| .csv | 支持 | 按文本读取 |
| 其他格式 | 跳过 | 保存元数据，不执行文件 |

### 9.4 ZIP 处理规则

- 保存原始 ZIP 附件。
- 为 ZIP 内文件创建子附件记录。
- 只解析支持的子文件类型。
- 不执行 ZIP 内任何文件。
- 超过大小或层级限制会标记失败并进入附件解析记录。

## 10. 常见业务场景

### 10.1 新订单信息完整

1. 销售发送完整订单邮件。
2. 系统同步邮件并分类为销售订单需求。
3. 系统抽取字段并通过初审。
4. 系统生成生产任务单并发送生产部门。
5. 销售收到收件回执。
6. 生产确认排产后任务关闭并进入周报统计。

### 10.2 新订单缺少交期

1. 销售发送订单邮件但未写交期。
2. 系统初审失败。
3. 销售收到补充请求邮件。
4. 异常队列出现 `ReviewNeedManual`。
5. 销售回复“期望交期：2026-05-20”。
6. 系统重新初审，通过后下达生产。

### 10.3 生产提出技术疑问

1. 生产回复任务单：“该产品包装要求不明确，请销售确认。”
2. 系统记录生产疑问并转发销售。
3. 销售回复补充包装要求。
4. 系统生成 V2 任务单并重新下达生产。
5. 生产确认排产后任务关闭。

### 10.4 外发失败

1. “外发”页面出现 Failed 状态。
2. 检查机器人邮箱密码、SMTP 主机、端口和网络。
3. 修复配置后，在失败邮件上点击“重试”。
4. 点击右上角“发送待发邮件”或等待自动 worker 发送。
5. 确认状态变为 Sent。

### 10.5 生产邮箱未配置

1. 销售订单通过初审。
2. 系统发现生产部门主送为空。
3. 异常队列出现 `RoutingMissing`。
4. 管理员进入“生产邮箱”配置主送邮箱。
5. 回到异常页面补字段或关闭后重新处理相关邮件。

## 11. 运维与数据管理

### 11.1 自动 worker

系统启动后，如 `MAIL_AUTO_WORKER_ENABLED=true`，会启动自动 worker。Worker 默认每 60 秒执行一次，主要动作：

- 处理入库队列。
- 发送自动流程外发邮件。
- 同步 IMAP 新邮件。

如果机器人邮箱密码未配置，worker 会跳过真实邮箱链路。

### 11.2 手动触发

管理员可通过页面右上角按钮手动触发：

- 同步邮箱：从 IMAP 拉取新邮件。
- 运行入库队列：处理待识别、待抽取、待生成任务单的队列。
- 发送待发邮件：发送外发队列中的 Pending 邮件。

### 11.3 数据保留

默认策略：

- 有效订单邮件、任务、附件和审计记录长期保留。
- 非目标邮件和临时记录默认保留 30 天。
- 附件存储预算默认 10GB。
- 清理前建议先执行“清理预览”。

### 11.4 备份

“立即备份”会生成包含 SQLite 数据和附件目录的 ZIP 备份。建议生产环境至少每周备份一次，并将备份文件转存到服务器外部安全位置。

### 11.5 审计

系统会记录关键操作，包括：

- 邮件收到和分类。
- 字段抽取和 LLM 兜底。
- 初审失败和补充请求。
- 任务草稿生成和任务下达。
- 生产疑问、销售答复和新版任务单。
- 生产确认和任务关闭。
- 外发失败和重试。
- 异常关闭和补字段恢复。
- 周报生成。
- 配置修改。

## 12. 本地启动与部署

### 12.1 本地 Python 启动

安装依赖：

```bash
python3 -m pip install -e ".[test]"
```

启动服务：

```bash
python3 -m uvicorn backend.app.main:app --reload --host 127.0.0.1 --port 8000
```

访问：

```text
http://127.0.0.1:8000
```

### 12.2 Docker 启动

复制配置文件：

```bash
cp .env.example .env
```

编辑 `.env`，填入管理员密码、邮箱密码、模型 API Key 等敏感配置。

启动：

```bash
docker compose up --build
```

默认端口：

```text
8000
```

Docker Compose 会将本地 `./data` 挂载到容器 `/app/data`，用于保存数据库、附件和备份。

## 13. 常见问题与处理

### 13.1 无法登录

检查：

- 用户名和密码是否正确。
- 是否修改了 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD`。
- `AUTH_SECRET` 是否稳定，频繁变化会导致旧会话失效。
- 浏览器是否禁用 Cookie。

### 13.2 同步邮箱无数据

检查：

- 机器人邮箱和密码是否配置。
- IMAP 主机和端口是否正确。
- 腾讯企业邮箱是否开启 IMAP/SMTP。
- 邮箱限频是否导致短时间内无法再次登录。
- 邮件是否已读或不在系统读取范围内。

### 13.3 邮件没有生成任务

检查：

- “邮件”页面分类是否为 `SalesOrderRequirement`。
- 是否缺少客户、产品、数量、交期等必填字段。
- 是否命中风险关键词。
- 是否生产邮箱未配置。
- 是否进入“异常”页面。
- 附件解析是否失败。

### 13.4 外发一直 Pending

检查：

- 自动 worker 是否启用。
- 机器人邮箱密码是否配置。
- SMTP 主机和端口是否正确。
- 是否需要手动点击“发送待发邮件”。
- 是否存在邮箱限频。

### 13.5 外发 Failed

处理：

1. 查看外发失败的邮件类型和收件人。
2. 检查邮箱密码、SMTP 配置和网络。
3. 修复配置后点击“重试”。
4. 再点击“发送待发邮件”或等待 worker。

### 13.6 附件解析失败

检查：

- 文件格式是否在支持范围内。
- PDF 是否为扫描件或图片 PDF。
- ZIP 是否超过大小或解压层级限制。
- Excel 或 Word 是否损坏或加密。

### 13.7 任务无法关联生产回复

建议生产部门直接回复系统发送的任务单邮件，并保留主题中的任务编号。若仍无法关联，系统会创建 `MailTaskLinkFailed` 或 `ProductionConfirmationTaskLinkFailed` 异常，需要商务人工判断。

### 13.8 LLM 兜底不可用

检查：

- “启用 LLM 兜底识别”是否开启。
- 模型 API Base、模型名称和 API Key 是否正确。
- 点击“测试模型”是否成功。
- 模型服务是否支持 OpenAI 兼容 Chat Completions 接口。

## 14. 安全注意事项

- 不要将邮箱密码、授权码、API Key 写入 README、文档、代码或提交记录。
- 生产环境必须修改默认 `admin/admin`。
- 建议使用独立机器人邮箱，不使用个人邮箱代发。
- 建议限制管理台访问来源，例如只允许内网访问。
- 定期备份数据和附件。
- 处理订单取消、已排产后变更、加急、客户未确认、价格待定等高风险场景时，应由商务负责人人工确认。
