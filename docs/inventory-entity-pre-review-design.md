# 库存管理、主体-仓库映射与预审库存判断 — 需求设计

---

## 一、管理台功能

### 1.1 库存导入与管理界面

```
库存管理页面：
  ├─ 下载模板（Excel）
  ├─ 上传Excel → 预览 → 确认导入
  ├─ 查看当前库存快照
  │   ├─ 按仓库筛选（武汉仓/欧洲仓/美国仓/...）
  │   ├─ 按物料编码/名称搜索
  │   └─ 显示：物料编码、名称、库存量、仓库、最近更新时间
  └─ 导入历史记录（谁、什么时候、导入了多少条）
```

**Excel模板格式：**
| 仓库 | 物料编码 | 物料名称 | 库存数量 | 备注 |
|------|---------|---------|---------|------|
| 欧洲仓 | 1010100001 | CR-scan 01上壳 | 500 | |
| 武汉仓 | 1010100002 | CR-scan下壳 | 200 | |

**说明：**
- 库存表不分主体，只分仓库（因为仓库不区分主体）
- 商务每周从涛哥拿到最新库存表，导入即可
- 导入覆盖该仓库的旧数据（以导入时间为准）

### 1.2 主体-仓库映射管理

```
主体-仓库映射页面：
  ┌────────────────────────────────────────┐
  │  主体编码   主体名称      关联仓库      操作  │
  ├────────────────────────────────────────┤
  │  SZ        深圳积木易搭   武汉仓        编辑  │
  │  HK        积木易搭(香港) 欧洲仓、美国仓  编辑  │
  │  US        美国主体       美国仓        编辑  │
  │  LU        卢森堡主体     欧洲仓        编辑  │
```

## 二、库存预审流程设计

### 2.1 完整决策树

```
CRM订单进入中台
  → 识别销售主体（从PI/CRM解析，如香港）
  → 识别需求物料+数量
  →
  → Step 1：查"主体对应仓库"的库存
  │   ├─ 香港主体 → 查欧洲仓 + 美国仓
  │   ├─ 够 → ✅ 正常制单, 备注发货仓
  │   └─ 不够 → 进入 Step 2
  │
  → Step 2：查"其他主体仓库"的库存
  │   ├─ 深圳(武汉仓)有货？/ 卢森堡(欧洲仓)有货？
  │   ├─ 有 → ⚠️ 可调货
  │   │   ├─ 标记"可跨主体调货"
  │   │   ├─ 通知商务确认调货方案
  │   │   └─ 商务确认后 → 走调货流程
  │   └─ 全都没有 → 进入 Step 3
  │
  → Step 3：所有仓库均缺货
      └─ ❌ 通知销售重新提交需求
          ├─ 告知：哪些物料缺货
          ├─ 告知：可能有货的仓库（如果有在途？）
          └─ 销售在CRM协调后重新提交
```

### 2.2 Step 1的详细逻辑

```
主体：香港
主体-仓库映射：欧洲仓、美国仓

查库存表：
  ├─ 欧洲仓：物料A=0台，物料B=50台
  ├─ 美国仓：物料A=20台，物料B=0台

订单需求：物料A*50台 + 物料B*30台
  ├─ 物料A：欧洲仓0 + 美国仓20 = 20台 < 50台 → 不够！
  ├─ 物料B：欧洲仓50 >= 30台 → 够
  └─ 结论：物料A不够，进入Step 2
```

### 2.3 Step 2的详细逻辑

```
Step 1发现物料A缺30台
  → 查其他主体仓库：
  ├─ 深圳(武汉仓)：物料A=100台 → 有货！可调货
  └─ 生成异常："香港主体订单缺物料A*30台，武汉仓有库存，是否调货？"
      通知对象：商务B
```

### 2.4 Step 3的详细逻辑

```
Step 2发现所有仓库全部缺货：
  → 通知销售（不是商务B，而是CRM订单的销售负责人）
  → 通知内容：
      "订单XXX（CRM订单号）的以下物料在所有仓库均库存不足：
       - 物料A（3D扫描仪标准版）：缺50台
       - 物料C（U盘16GB）：缺100台
       目前可能有货的仓库：无（有在途？待确认）
       请协调后重新在CRM中提交需求。"
  → 销售在CRM中修改需求（改数量/改物料/取消）后重新提交
  → 中台重新预审
```

---

## 三、通知对象区分

| 阶段 | 通知对象 | 原因 |
|------|---------|------|
| Step 1：主体仓库够 | 不需要通知 | 走正常流程 |
| Step 2：其他主体仓库有货 | **商务B** | B需要协调调货方案 |
| Step 3：所有仓库缺货 | **销售** | 销售决定改需求还是取消 |
